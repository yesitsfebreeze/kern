## MCP server

The server side of the Model Context Protocol: the `initialize` handshake +
`initialized` notification, `tools/list`, `tools/call`. JSON-RPC 2.0
newline-delimited UTF-8 frames вЂ” one complete JSON object per line, embedded
newlines not permitted (**not** LSP's `Content-Length` framing). The wire
format is an implementation detail hidden inside `mod server`; `serve_stdio`
/ `serve_rw` / `serve_http` run the server side.

## Typed-RPC stack

A codec-framed `typed::Channel` over a `typed::Adapter` (in-proc pair or the
per-cwd local socket), with a pluggable `Codec` (`JsonEnvelope`). The
`service!` macro (re-exported from [`trnsprt-macros`](macros/)) generates a
trait, a client, and a serve loop per service from a single declaration. The
two services:

- **`kern_rpc`** (`KernRpc`) вЂ” the kern daemon RPC surface, consumed by the
  CLI and other local clients over the per-cwd socket.
- **`hub_rpc`** (`HubRpc`) вЂ” the hub's control surface.

`search` holds the shared entity/edge DTOs their wire shapes overlap on.

## Extending

- New transports в†’ implement a `typed::Adapter`.
- New services в†’ one `service!` declaration generates trait + client + serve
