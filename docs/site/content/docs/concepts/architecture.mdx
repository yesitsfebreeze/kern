This is the one document that explains what kern does, the order it does it in,
and the reason each part is shaped that way. First half reads without knowing
Rust. Second half carries `file:line` anchors for contributors.

---

## Part 1 — What kern is

kern is a **memory daemon**. One long-running process per working directory owns
a knowledge graph. You feed it durable facts — an agent calls MCP `ingest`, or a
transcript is dropped into `.kern/intake/` — and from there it keeps itself small
on its own and serves the right context back at recall time. It captures no
session automatically; the write is always a caller's call.

The main feature — the one everything else exists to serve — is this:

> **Recall costs no LLM call.** The default query path is a sub-millisecond graph
> walk. No network hop, no model, no cloud.

Every other design choice follows from paying that bill somewhere else.

If recall must be cheap, then all the expensive thinking has to happen *before*
the query arrives. So kern spends an LLM at write time (distilling a session
into typed claims), spends CPU in a background tick (re-embedding, clustering,
deciding what is stale), and leaves the read path with nothing to do but walk
structure that is already there.

That single trade explains the whole system:

| Decision | Why |
|---|---|
| One daemon per cwd, always running | A background tick needs somewhere to live. Batch jobs can't maintain a graph. |
| Distill at ingest, not at query | The LLM cost is paid once per session, not once per lookup. |
| A graph, not a vector bag | Reason edges are precomputed *why*. Walking them at query time is free; deriving them isn't. |
| IDs are content hashes | Identical text is the same node everywhere. Merge becomes set union — no coordinator needed to federate. |
| Heat + decay + GC | Nobody prunes by hand, so the graph has to decide what to forget itself. |
| All-internal, no pluggable backend | Mounting an external store adds a network hop to the one path that must stay sub-ms. |
| Fail open everywhere | Memory that breaks a session is worse than memory that misses. Intake and recall no-op on any error. |

## The flow, in one line

```
MCP ingest ─────────────────────► typed claims ─┐
.kern/intake/ drop → distill (LLM) → typed claims ┴→ graph → tick → recall
```

Two caller-driven write entries, no automatic capture: an agent calls MCP
`ingest` to store a claim directly, or a transcript dropped into `.kern/intake/`
is distilled into claims. Both land in the same graph.

Four independent loops, not one pipeline:

1. **Write loop** — a claim arrives (via MCP `ingest`) or a dropped transcript is distilled, and is placed in the graph.
2. **Read loop** — a query arrives, seeds candidates, expands over edges, ranks, returns.
3. **Maintenance loop** — a tick fires every 60s: deposit heat, cluster, re-embed, evict, persist.
4. **Federation loop** (off by default) — gossip merges entity bodies with peers.

They share one `Arc<RwLock<GraphGnn>>` and nothing else. That is the whole
concurrency story.

## The unit of storage

Two things live in the graph.

- **Entities** ("thoughts", "claims") — a distilled fact, decision, or
  preference. Typed, carries a Beta-distribution confidence, a heat value, a
  vector, and a bi-temporal window.
- **Reasons** (edges) — a *justified* link between two entities. Not a
  similarity score: an actual labeled relation, with its own vector and score.

Entities live inside **kerns** — a tree of buckets. The root is a pure
dispatcher; its named children are **gravitons**, focus attractors you seed once
("decisions", "project state", "preferences"). Ingest routes a new claim to the
graviton it is closest to, scaled by that graviton's `mass`. Anything matching
nothing lands in `generic`, and a dense `generic` cluster eventually gets
promoted into a graviton of its own.

## Nothing is deleted

An updated or contradicted claim is **superseded**, not removed. The old
revision stays, stamped with when it stopped being true. A query can ask `as_of`
a past instant, or `include_history` to walk the supersede chain back.

The update-vs-contradiction classification runs in the **background tick**, never
on the read path — because that decision needs an LLM, and the read path may not
have one.

## Part 2 — The working order, with anchors

### Boot

`src/main.rs:10` is the only real binary. Order matters at every step:

1. Tracing to **stderr** (`main.rs:12`) — stdout is reserved for JSON-RPC.
2. Tokio multi-thread runtime, hard floor of 4 workers (`main.rs:22-27`) — blocking bridges each pin a worker.
3. Re-pin cwd to the nearest ancestor holding `.kern` (`main.rs:33-42`).
4. Load + validate config (`main.rs:43-50`).
5. `Cli::parse()` (`main.rs:51`), then dispatch (`main.rs:53-57`).

**Bare `kern` with no subcommand is the daemon.** Only `Daemon` and "nothing"
reach `run_server`; everything else goes through the single command dispatch
table (`commands.rs:441-553`).

`bootstrap` (`commands.rs:555-665`), in order and for reasons:

| Step | Anchor | Why here |
|---|---|---|
| `maybe_self_heal_store` | `:558` | Compaction swaps `data.mdb`; only before any env open is the dir held exclusively. |
| `spawn_watchdog` | `:560` | OS thread, force-exits on ~30s async stall (`:744-782`) so a peer can take the hub. |
| Build LLM client | `:561-577` | Embed/answer models always come from config, never the CLI (`:571`). |
| `spawn_keepalive` | `:579` | 240s ping — Ollama unloads a model at ~5min idle. |
| **`Registry::open`** | `:600` → `store.rs:57` | The real assembly point: graph load, the single guarded `save_fn`, tick queue, deferral closures, ingest `Worker`, `tick::start`, `enqueue_all`. |
| Reap empty kerns | `:614` | |
| MCP server, watcher, intake, gossip, maintenance tick | `:630-658` | |

`run_server` (`:667`) binds `kern.sock` **synchronously** (`:687-712`) so a
second daemon short-circuits on `AlreadyRunning` before spinning up scaffolding.

> Divergence: standalone `kern mcp` (`commands/mcp_cmd.rs:240-326`) builds its
> own server with `tick::start` but **no maintenance tick and no gossip**
> (`spawn_maintenance_tick` appears only at `commands.rs:658`).

**One dispatch core.** Every surface — MCP, RPC, CLI — routes through the same
dispatch. A second copy is a repo law violation, not a style preference: two
copies drift, and the drifted one is always the one in production.

### Write path

1. **Entry.** MCP sync (`tools_mutate.rs:201`) blocks on the worker; async
   (`:239`) writes a `DirectJob` into the intake dir; fallback (`:267`)
   is a RAM-only enqueue. CLI at `commands/ingest_cmd.rs:15`, watcher at
   `ingest/file_watcher.rs:75`.
2. **Queue.** `mpsc::channel(64)` (`ingest/worker.rs:43`), single consumer, not
   batched (`:123`). `enqueue` detaches — no backpressure (`:75-77`); `run`
   awaits a oneshot — backpressure yes.
3. **Distill** (transcript path only), `ingest/distill.rs:21-51`. One
   `/api/chat` call, `stream:false`, `think:false`, `num_ctx` 8192
   (`llm.rs:512`), 600s timeout (`:517`). The prompt (`distill.rs:26-44`) asks
   for a bare JSON array of `{text,kind}`. The long timeout is deliberate: an
   LLM outage must **queue**, never lose the delta.
4. **Split → embed doc → doc dedup → embed chunks → chunk dedup → place**
   (`worker.rs:203-271`).
5. **Placement** (`base/accept.rs:24-30`): dedup, route from root by graviton
   match, commit. Root never accepts directly — no match falls to `generic`
   (`:128-136`). Below root, `effective_distance = cosine_distance / mass`
   (`:20-22`) against `ACCEPT_FLOOR = 0.5`.
6. **Persist** after *every* job (`worker.rs:133-135`) via
   `save_graph_guarded` (`commands.rs:281`): snapshot under a read guard,
   drop the guard, `flush_guarded` compares the on-disk epoch, `RefusedStale`
   absorbs and retries, 5 rounds. This is what stops a stale in-memory snapshot
   from overwriting newer disk state.

### Read path

`tool_query` (`tools_query.rs:114`) → cache probe → embed → `query_locked`
(`retrieval/answer.rs:297`). The lock choreography is the point:

1. **HyDE first, lock-free** (`:310`) — it touches no graph, so it must not hold
   a lock.
2. **Retrieve under one read lock** (`:314-318`). The epoch is captured under
   the *same* lock, so a concurrent write makes the cache stamp born-stale.
3. **Rerank + answer with the lock released** (`:320-335`) — a slow LLM must
   never pin the read lock, or it trips the 30s watchdog.

`retrieve_profiled` (`answer.rs:172`) stage order:

```
seed_important (O(N), once)  :185
dense seed                   :186
hybrid BM25 fuse             :189-197
expand (beam over edges)     :211
merge (log-sum-exp)          :213
apply_boosts                 :217
apply_gravity                :218
active filter                :220-225   ← must precede truncation
filter_delivery              :226
dedup_by_section             :232
mmr                          :234
```

**Ranking signals** (`retrieval/score.rs:82-93`):
`score * confidence + qbst_boost + fact_bonus`, where `qbst` blends access count
and recency (capped 0.1, `config/retrieval.rs:88-91`). Mode weights (`:104-114`)
— hybrid is content 0.50 / reason 0.30 / edge 0.20. Per-neighbor score is
`w.content*cos + w.reason*cos + w.edge*edge_score`, and `edge_score` adds a log
traversal boost (`expand.rs:305-329`).

Defaults on: lexical, mmr (λ 0.75), rerank, hyde, pagerank.

### Maintenance tick

`spawn_maintenance_tick` (`commands.rs:942-979`), every `tick.interval_secs`
(default 60, `base/constants.rs:27`; 0 disables). Per beat, in order:

1. `reconcile_if_stale` (`:961`) — **first**, or a per-kern persist writes stale
   rows over newer disk.
2. Read root id, `pulse` under the write lock (`:968`).
3. Broadcast pulse (`:970`), `enqueue_all` (`:973`), `snapshot_if_dirty` (`:976`).

Tasks drain one at a time off an mpsc queue (`tick.rs:37-52`); 13 kinds
dispatched at `tick.rs:54-74`.

`do_cluster` (`tick.rs:77-137`) is the ordering lesson worth reading: select
clusters → spawn children → collect follow-ups → evict empty → **drop the lock**
(`:110`) → enqueue Name/Cluster/Enrich/ResolveQuestion → **persist children
before the parent** (`:126-135`). Parent-first plus a crash erases migrated
entities; child-first merely duplicates them briefly. GnnPropagate runs only if
structural work actually happened (`:124-137`).

**Heat** (`base/heat.rs:25-38`) is exponential — `heat * exp(-ln2 * dt /
half_life)`, half-life 7 days (`:18`) — and decays **lazily on read**, not per
tick, so an idle graph costs nothing.

**Bi-temporal fields** (`base/types.rs:290-297`): `valid_from`/`valid_to` are
world time, `invalidated_at` is transaction time. All three are
`#[serde(skip)]` — load-bearing, it keeps bincode byte-identical to
pre-temporal snapshots; `StoredKern`'s side-map persists them instead. Supersede
is a status flip plus `superseded_by` (`accept.rs:346`); superseded rows are
filtered at delivery (`score.rs:119`) but stay readable via `include_history` /
`as_of`.

### Federation (off by default)

Enabled only via config (`config/gossip.rs:62`); `start_gossip`
(`commands.rs:858-940`) spawns five 30s loops: heartbeat, announce, entity_sync,
delta_flush, discovery.

**Send.** A mutation calls `push_delta` (`base/graph.rs:404`), coalesced by
`(object_id, target)` — correct *because* counters carry an absolute slot total,
not an increment (`gossip/types.rs:113-114`). Within 30s `start_delta_flush`
(`handler.rs:191-232`) drains and broadcasts. `broadcast` (`node.rs:132`)
seen-checks, then `forward` shuffles peers and truncates to fanout 3
(`node.rs:245-264`). One fresh TCP connection per message
(`transport.rs:37`) — no pooling. Frame: BE u32 length + bincode
(`transport.rs:9-20`).

**Receive.** Accept → decode with a 4 MiB pre-allocation cap
(`transport.rs:27`) → Fetch short-circuit → seen-dedup (`node.rs:204`) → peer
learning from `origin` (`:208-210`) → routing hint → handler → re-forward
regardless of outcome (`:220`).

**CRDT rules** (`crdt.rs`, `base/merge.rs`). Counters merge per-slot max.
ReasonScore and ValidUntil are LWW on `(lamport, producer)`, compared through
the shared `crdt::lww_wins`. `conf_alpha`, `conf_beta`, `unlinked_count` **and
`statements`** are **deliberately never imported** — a max-join on confidence is
an irreversible poisoning pin, and statements are content-addressed, so a merge
can only ever admit text whose hash is not the id it arrived under. Cross-kern
id claims are rejected as hijacks (`merge.rs:167-176`).

## Load-bearing invariants

- **Content-addressed IDs** — `id = sha256(text)`; equal ids ⇒ identical
  content. Dedup updates metadata only, never text or vector → CRDT-safe.
- **Confidence is replica-local** — Beta(α,β) is never merged from a remote.
  Only access/traversal GCounters federate.
- **Reason hosting** — an edge lives in its `from` kern; `to_kern_id` /
  `to_net_id` stamp cross-kern and cross-network targets.
- **Hybrid dense score** — `0.4·entity_idx + 0.6·gnn_entity_idx` wherever search
  runs. The GNN is a *background re-embedder* feeding this seed-time blend, never
  a query-time reranker (the rerank stage is the optional LLM one).
- **Heat → GC** — access deposits heat, pulse re-deposits on root-reachable
  entities. Reaped when `heat<0.01 AND stale>7d AND kind ∉ {Fact, Document}`,
  where the staleness clock is `accessed_at` falling back to `created_at`. Heat
  is decayed *at the comparison*, not written back — a predicate that read the
  stored value would let a once-hot, never-touched entity evade eviction forever.
- **Single-writer store** — one LMDB env per data dir; readers never block,
  writers serialize, `flush_guarded` epoch-checks against stale overwrite.
- **Global mutation epoch** (`base/graph.rs:84-85`) — must stay global;
  per-kern versions are unsound under HyDE.
- **Watchdog** — an OS thread force-exits (101) on a 30s async stall so a peer
  can take the hub.

Notes: `diskann.rs` backs the optional `VectorBackend::Disk` spill for the entity
index (`[graph] disk_threshold`, **off by default** — resident HNSW is the
default path). `[graph] max_kerns` defaults to `usize::MAX` (cap off); empty-kern
GC keeps it from bloating.
