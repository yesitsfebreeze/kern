title: "The graph: nodes, edges, claims"

Memory is a graph: *thoughts* connected by *justified edges*, held in a tree of kerns, persisted to one LMDB file. The data model every other page assumes.

## Why a graph, not a vector store

Similarity is symmetric, unlabelled, query-time only. A kern edge is a stored, typed, directional assertion. Buys:

- **Traversal** ‚Äî retrieval expands from seeds along edges; a non-matching thought is reachable through one that matches. (Reaching is not yet ranking ‚Äî caveat in [Retrieval](./retrieval.mdx).)
- **Explanation** ‚Äî edges carry a written reason; results return the chain walked.
- **Non-similar relations** ‚Äî "deadline is March" / "deadline is April": near-identical vectors, opposite meanings ‚Äî stored explicitly as a supersede edge. See [Time](./time.mdx).

## Thoughts

One atomic unit of memory; `kind` decides treatment:

| Kind | Meaning |
| --- | --- |
| `Fact` | Durable assertion from a trusted source. Never auto-forgotten while active. |
| `Claim` | The default. Distilled from a session; may or may not survive. |
| `Document` | A chunk of an ingested source document. |
| `Question` / `Conclusion` | Internal-only ‚Äî seeded and resolved by the system itself. |

Callers may supply only `Fact` and `Claim` (rest rejected at the MCP surface); `Fact` additionally requires a trusted source (user or agent, not an arbitrary file).

Also on a thought:

- **Confidence** ‚Äî Beta distribution; corroborations and contradictions are observations, so a claim seen five times outranks one seen once.
- **Heat + access counts** ‚Äî drive eviction; see [Heat](./heat-and-compaction.mdx).
- **Two vectors** ‚Äî content embedding + structural embedding recomputed from the neighbourhood; retrieval blends both.
- **Validity window + status** (`Active`/`Superseded`) ‚Äî bi-temporal machinery; see [Time](./time.mdx).
- **Provenance** ‚Äî typed source: file, ticket, session, agent, or inline.

## Justified edges

`from` ‚Üí `to`, with a kind, optional explanatory text, own vector:

| Kind | Meaning |
| --- | --- |
| `Similarity` | Attached at commit to the nearest neighbour ‚Äî no thought is isolated. |
| `Provenance` | Thought ‚Üí its source document. |
| `Supersedes` | New revision ‚Üí the one it replaced. |
| `Rephrase` | Same thought, different wording; alternate phrasing lives in the edge text. |
| `Question` / `Spawn` / `Ratification` | Question linkage, kern lineage, corroboration. |

## Claims, not raw transcripts

A transcript in `.kern/intake/` is never stored as memory. One LLM pass distills typed claims ‚Äî atomic (independently retrievable, supersedable), typed at birth (kind-aware retention), deduplicated against everything known. Empty distillation archives the delta; no output at all (outage) leaves it queued for retry ‚Äî nothing silently lost. See [Acceptance](./acceptance.mdx).

## Identity: content hashes

Id = SHA-256 of the text.

- Identical content = same node everywhere, forever. Federation merge is set union ‚Äî no coordinator.
- **Text and vector never overwritten under an existing id** ‚Äî a near-duplicate with different phrasing gets a `Rephrase` edge; the tick decides contradiction later.
- A duplicate (cosine above the preset's dedup threshold ‚Äî 0.98 on default `relaxed`) is not discarded: it bumps the existing thought's confidence.

## The kern tree

Thoughts live in `Kern` containers in a tree. A named kern has a **graviton** ‚Äî label, centroid vector, mass (default 1.0). Routing descends by `cosine_distance / mass` ‚Äî a heavier graviton attracts more strongly. The root never holds thoughts; no match falls to a `generic` catch-all. See [Acceptance](./acceptance.mdx).

## Storage

One LMDB environment per data directory, three tables: kern rows, cold tier, graph metadata. Values are versioned zstd ‚Äî an older reader rejects a newer file rather than misparsing it.

- **Int8-quantized vectors** on disk (~4√ó smaller); structural vectors recomputed, not persisted.
- **Epoch-guarded writes** ‚Äî a stale in-memory snapshot never overwrites newer disk state.
- **Two tiers** ‚Äî GC spills to the cold tier *before* dropping from memory; a failed spill keeps it hot. Cold tier: capped (50,000 entries), unindexed tail, searched only when the hot graph comes up short. Active `Fact`s and `Document`s never evicted. Past the cap the oldest rows genuinely go ‚Äî each drop counted by `health`.
