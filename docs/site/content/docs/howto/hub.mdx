---
title: "The hub"
---

A machine-level supervisor that owns detached kern daemons. It is not a server
of graphs — each daemon still owns its own per-cwd graph and socket — it is the
one process a host runs to spawn, find, unload, and stop them.

## What it does

- **Resolve** a cwd to the daemon serving it (or spawns one and reports the
  socket path), so `kern mcp` and other clients never have to race the bind
  themselves.
- **Status** — every hub-owned daemon, alive or parked, with its pid and idle
  time.
- **Unload** an idle hub-owned daemon from memory without stopping the hub.
- **Merge** — `kern hub merge <src> <dst>` performs an offline CRDT union of
  `src`'s graph into `dst`'s. **Both daemons are stopped first**; `src` is read
  only and never written. Use it to consolidate a project's graph into another.
- **Stop** — ends the hub daemon; **nodes stay up**. A node without a hub is
  just a daemon with a socket, still serving its graph.

## Auto-start and auto-restart

`[hub]` config (`kern.toml`):

| Key | Default | Effect |
| --- | --- | --- |
| `auto_start` | `true` | `kern mcp` spawn-attaches a detached hub when none answers, the same way it auto-spawns a project daemon. `false` = hub is opt-in via `kern hub`, and the direct-connect fallback still works. |
| `auto_restart` | `true` | A client attaching to a daemon built from a different binary, or booted against a different config, restarts that daemon before proxying. Without this a long-lived detached daemon serves stale code and stale config indefinitely — the failure that makes every shipped fix look like it did nothing. |

## Idle unload

A hub-owned daemon idle for `--idle-unload-secs` (default **1800 s** = 30 min) is
unloaded and re-spawned on next contact; `0` disables. The root daemon is never
auto-unloaded. Idle time is stamped in the MCP dispatch core and every typed
RPC method, and the reaper double-checks under the root lock — a node that
touched anything between the sweep scheduling and the sweep running stays.

## Where state lives

- Hub socket: `$XDG_RUNTIME_DIR/kern-hub.sock` (or `/tmp/kern-hub-<user>.sock`
  on Unix; `\\.\pipe\kern-hub-<user>` on Windows).
- Node handles live next to the hub daemon's; the per-project graph still lives
  under `<project>/.kern/data` — the hub never owns graph data.

## Commands

```bash
kern hub                       # start the hub (foreground)
kern hub status                # list nodes
kern hub resolve               # resolve the cwd's daemon
kern hub resolve --root /path  # resolve another project
kern hub unload                # unload the cwd's idle node
kern hub merge <src> <dst>     # offline CRDT union (stops both)
kern hub --idle-unload-secs 0  # disable idle unload
kern hub stop                  # stop the hub (nodes stay up)
```
