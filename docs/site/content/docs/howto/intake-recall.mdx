title: "Intake & recall"
---

Start a daemon, seed the graph, drop an intake delta, confirm it landed.

## Before you start

`kern` on PATH, local [Ollama](https://ollama.com) with both default models:

```bash
ollama pull qwen3-embedding:0.6b   # embeddings
ollama pull granite4:3b            # distillation and reasoning (write path)
```

<Callout type="warn">
Embedding dimension locks into the graph on first ingest; switching models later requires `kern reembed`. Not silent: store stamps its model, `kern health` prints `MISMATCH` beside `embed:`, wrong-dimension query vectors are dropped and counted.
</Callout>

## 1. Opt the project in

```bash
cd /path/to/your/project
mkdir -p .kern
```

Pins to the nearest ancestor with `.git` or `.kern`. No config file required ŌĆö create `.kern/kern.toml` only to override a default.

## 2. Start the daemon

Let your MCP client start it ŌĆö `kern mcp` attaches to a running daemon, auto-spawns a detached one if none answers:

```json
{
  "mcpServers": {
    "kern": { "command": "kern", "args": ["mcp"] }
  }
}
```

The daemon writes this entry into `.mcp.json` on startup if absent. To watch the first run (logs on stderr only; stdout is reserved for JSON-RPC):

```bash
RUST_LOG=kern=info kern --daemon
```

## 3. Seed gravitons and claim kinds

Once per project, via MCP against the running daemon (the CLI can race a live daemon).

- `graviton` tool, `action: "add"` per focus area ŌĆö e.g. *decisions*, *project state*, *preferences*.
- Seed `text`: **example statements, one per line** ŌĆö each line embedded separately, mean-pooled into the pull vector; routes far better than an abstract description.
- Optional `mass` (default `1.0`): stronger attraction.
- Unmatched claims ŌåÆ `generic`. See [Acceptance & routing](../concepts/acceptance.mdx).
- Built-in claim kinds: `preference`, `decision`, `project`, `fact`, `code-fact`, `reference`, `procedural`. Extras via `claim_kind`; registered kinds are offered to the distillation LLM alongside built-ins.

CLI equivalents, daemon stopped:

```bash
kern graviton add decisions "architectural choices and their rationale" --mass 1.5
kern graviton list
kern claim-kind add incident "a production incident: what broke and the fix"
```

## 4. Drop a session into the intake

Anything writing a `.txt` delta into `.kern/intake/` feeds kern ŌĆö client hook, shell wrapper, your own script:

```bash
cat > .kern/intake/session-$(date +%s).txt <<'EOF'
user: we're standardizing on tabs for indentation across the repo
assistant: noted ŌĆö updated the editorconfig
user: also the deploy key lives in the ops vault, not in CI secrets
EOF
```

- Daemon polls every 5s, distills each delta in one LLM pass, ingests the typed claims.
- Moves to `.kern/intake/done/` only once every claim committed; LLM outage ŌåÆ stays put, retried.
- Archived deltas pruned after seven days.
- Synchronous, already-typed ingest (skips distillation): `ingest` MCP tool or `kern ingest`.

## 5. Recall in the next session

`query` MCP tool: `mode`, `kind`, `source`, time ranges, `min_conf`, `as_of` (point-in-time), `include_history` (supersede chains). Returns scored passages, edges, chains ŌĆö the agent synthesizes. See [Retrieval](../concepts/retrieval.mdx).

Shell: `kern search "indentation" --k 5` or `kern query "what did we decide about indentation"`.

## What a healthy graph looks like

```
data_dir:    /path/to/project/.kern/data
gravitons:     decisions, project state, preferences
kerns:       5
thoughts:    47 (unnamed: 3)
reasons:     91
claim kinds: 0
embed:       qwen3-embedding:0.6b (dim 1024)
evicted:     0 cold rows dropped
tick:        queue 0 | done 312 | avg 8 ms
degraded:    0 panics | 0 failures
  kern:decisions  thoughts:12  reasons:24
  kern:generic  thoughts:9  reasons:15
```

- `reasons` well above `thoughts` ŌĆö claims connecting, not just arriving.
- `gravitons` non-empty ŌĆö `(none)` means seeding never happened; everything piles into `generic`.
- `unnamed` small relative to `thoughts` ŌĆö large share means gravitons don't cover the content.
- `done/` filling while `.kern/intake/` stays near empty ŌĆö drain keeping pace.
- Non-zero panics/failures ŌĆö something dying; the line beneath names the task.

## When it does not work

<Mermaid
  chart={`
flowchart TD
    A["nothing in the graph"] --> B{".txt files piling up in .kern/intake/?"}
    B -->|"no, and done/ is empty"| C["nothing is writing deltas ŌĆö kern ships no writer; wire your harness/script to drop files, or use MCP ingest instead"]
    B -->|"yes, never drained"| D{"daemon running?"}
    D -->|"no"| E["start it: kern --daemon"]
    D -->|"yes"| F{"reason endpoint reachable?"}
    F -->|"no"| G["fix [reason].url / pull the model"]
    F -->|"yes"| H["model is replying prose, not JSON ŌĆö every delta requeues, so the drain never finishes"]
    B -->|"drained, done/ filling"| I{"kern health shows thoughts?"}
    I -->|"no"| J["embed endpoint failing ŌĆö check logs for kern.ingest"]
    I -->|"yes"| K["graph is fine; recall is the problem"]
`}
/>

- **LLM endpoint unreachable.** Transient: deltas queue and retry forever. Confirm `curl http://localhost:11434/api/tags`; look for `kern.ingest.intake` warnings mentioning "distill got no LLM output". WSL (NAT) with Ollama on the Windows host: `localhost` resolves inside the VM ŌĆö pin the URL to the host gateway (`ip route show default | awk '{print $3}'`) in `.kern/kern.toml`. Blanking both `[reason].url` and `[embed].url` disables distillation; the daemon logs a warning.
- **Nothing landing.** Check in order: `.kern/` exists where the daemon pinned; `[intake].enabled` is `true`; something is writing files. Non-UTF-8 ŌåÆ `.kern/intake/failed/`. Endpoint answers but drain never finishes: prose-not-JSON (see chart); `done/` empty while `.kern/intake/` grows is the tell.
- **Recall empty.** `kern health` shows content: check `MISMATCH` beside `embed:` (fix: `kern reembed`). Otherwise widen the query ŌĆö `min_conf` gates out fresh claims, which start at 0.6 until reinforced.

## Tuning

```toml
[intake]
enabled = true              # set false to opt out of self-learning entirely
dir = ".kern/intake"       # intake directory, cwd-relative
poll_secs = 5               # drain cadence
done_retention_secs = 604800
```

`poll_secs = 0` rejected by the validator (busy-loop).
