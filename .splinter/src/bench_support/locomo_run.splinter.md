# src/bench_support/locomo_run.rs — commentary

Metrics aggregated per run (moved from the `//!` doc): per-category quality = token-F1, ROUGE-L, LLM-judge (abstention for the adversarial category); retrieved-context size as a token-efficiency proxy; query latency p50/p95.

- `run_eval`: the answerer + embedder share one client (kern's reason endpoint doubles as the answerer). `.for_eval(seed)` means reason calls run on GPU — they are the eval workload, not background distillation — and ollama sampling is seeded so multi-seed runs give error bars. The judge is a separate client so the judge model can differ from the answerer; the answerer/distiller keep the model-default temperature (the seed makes that sampling reproducible per run) while the judge is pinned to temperature 0.
- `EvalContext`: introduced so `eval_sample` takes one context plus per-sample inputs instead of nine positional args (dropped its `too_many_arguments` allow).

Second-pass migration: this file was already close to the bar. Kept inline as one-line traps: judge-at-temperature-0 (verdicts must not carry sampling noise), fresh-graph-per-dialogue (LoCoMo personas are independent), the no-tick-loop/no-defer-hook explanation for the three `None`s passed to `Worker::new`, the two outage `continue`s, and `eval_sample`'s two-phase answer-then-judge doc — that one is a real workaround with a reason (answerer and judge cannot share an 8 GB GPU, so interleaving would pay a model reload per probe). Deletions: `ingest_sample`'s "Returns the number of claims ingested" (restated the return type), `block_on_embed`'s bridge name (the call is right there), and two test comments duplicating their test names (`distill_locomo_llm_outage_returns_none`, `distill_locomo_malformed_json_returns_empty_claims`).
