# src/trnsprt/src/typed/mod.rs — commentary

Landed as Phase 1, additive — lives alongside the legacy MCP `Client`/`Transport`/`InProcServer` plumbing without touching it. Service traits are generated by the `service!` proc macro (re-exported from `trnsprt`); the macro emits a client struct + a `serve_<service>` async fn that loop over a `Channel`.

Composing a `Channel` (in-process pair; swap the adapter for `TcpAdapter`/`LocalAdapter` for cross-process):

```rust
let (client_side, server_side) = InprocAdapter::pair();
let client = Channel::new(client_side, JsonEnvelopeCodec::new());
let server = Channel::new(server_side, JsonEnvelopeCodec::new());
// Hand `client` to a generated `FooClient::new(client)`, run
// `serve_foo(server, handler)` on a task.
```

Second-pass migration: module doc compressed to the three-layer inventory (Adapter/Codec/Channel). The usage example and Phase-1-additive context are preserved above.
