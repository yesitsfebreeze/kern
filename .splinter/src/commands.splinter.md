# src/commands.rs — commentary

- `embed_fn`: one shared definition for the tick worker and the `profile` command so the call sites can't drift; the standalone MCP path deliberately uses a distinct runtime-handle variant and is not routed through this.
- `maybe_self_heal_store` / `bootstrap` self-heal: LMDB never returns freed pages to the OS. The historical unnamed-kern spawn runaway (since fixed) left graphs with tens of thousands of empty kerns and multi-GB `data.mdb` files that every load must mmap and the startup reap alone can't shrink; retrieval and tick are O(loaded kerns), so the bloat taxed every request. Guarded by `SELF_HEAL_BLOAT_BYTES` (512 MiB) so a normal small graph pays nothing at boot. The in-`bootstrap` reap makes every restart converge to a clean graph.
- `spawn_watchdog`: exists because a wedged daemon (graph-lock deadlock, panic loop, or all workers pinned in blocking LLM calls) keeps holding kern.sock and stops serving with no visible failure. The 30s threshold sits far above any single LLM call — normal blocking work pins one worker, never the time driver, so the beat keeps advancing; only a TOTAL stall trips it. The stall message goes to stderr because stderr lands in daemon.err.log, keeping the next wedge visible.
- `bootstrap` reason-endpoint fallback (CLI flag wins, else `[reason]` config which falls back to `[embed]`): before this fallback the daemon ignored configured reasoning entirely — distillation/edge-proposal silently disabled unless `--reason-url` was passed.
- `bootstrap` embed endpoint: historical bug — the `--daemon` dispatch hardcodes `cli.embed_*` to `DEFAULT_EMBED_*`, so reading the CLI flags here ignored `[embed]` in kern.toml; the daemon embedded with the default model even when the graph was built with another, and the dimension mismatch degenerated every cosine. Hence `server_llm_client` takes answer/embed from config only.
- `bootstrap` tick_llm gate: the ungated `Some` churn (no-op Name re-enqueue, SeedQuestions no-ops) used to mask the misconfiguration instead of surfacing the capture path's loud "no reason LLM" warning.
- `bootstrap` mcp_server: one Arc shared by three consumers — the kern_rpc handler, the SSE/HTTP transport, and stdio.
- `run_server`: registers kern in `.claude/settings.json` at startup so Claude Code picks it up.
- `spawn_keepalive`: keepalive cadence history — cold reload is ~7s for qwen3-embedding, more for the 4b answer model; the 4-min ping is cheap and self-contained, no dependency on OLLAMA_KEEP_ALIVE.
- `spawn_file_watcher`: originated as Slice O. `effective_roots` applies the documented "empty roots → cwd" rule.
- `spawn_capture`: two file-mediated loops — capture drain (intake deltas → distill → enqueue → archive) and digest writer (periodic snapshot of purpose + hot thoughts).
- `start_gossip` ledger cap: `node.ledger.set_max_entries(cfg.graph.max_ledger_entries)` exists because the config knob was previously dead — the routing/thought ledger always used the hardcoded `DEFAULT_LEDGER_CAP`.
- `spawn_maintenance_tick`: the interval snapshot bounds the crash-loss window for any mutation whose event-driven save never ran (task crashed pre-Persist, SIGTERM before shutdown flush) to one tick interval.
