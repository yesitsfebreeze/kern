# src/tick/tasks.rs — commentary

- `strip_name_prefixes`: kept pure so the parsing is unit-testable apart from `do_name`'s graph/LLM plumbing.
- `do_seed_questions`: relocated from the ingest worker's `place_chunks`, where it was one BLOCKING reason-LLM call per placed chunk on the commit path — measured live: a one-line sync ingest queued 69.7 minutes behind LLM-bound jobs. The worker now enqueues a `SeedQuestions` tick task per placed chunk via its defer hook; the tick already serializes LLM work off the interactive path.
- `do_classify_contradiction`: the background half of contradiction-driven invalidation — the ingest worker (which carries no reason-LLM) records a same-kind, different-text near-dup as a `Rephrase` edge and defers here via its dedup hook. On a `Supersede` verdict the new phrasing is embedded, materialized as a fresh Active revision, and the stored claim invalidated (status/superseded_by/valid_to/invalidated_at + ANN eviction + `Supersedes` edge). The new revision uses the same single-source Entity builder as the ingest path (`build_chunk_entity`; kind carried from the stored claim, no external_id — a contradiction, not a re-ingest).
- `do_name`: `promote_to_root_if_generic` is emergent promotion — a dense cluster that crystallized inside the `generic` catch-all becomes a first-class anchor directly under the root, so future matching memories route to it instead of generic.

## Second-pass migration:
- Lock discipline (moved from inline on do_seed_questions / do_classify_contradiction / do_resolve / do_reembed): the shared pattern is snapshot under a read guard → LLM/embed/ANN-search UNLOCKED → mutate under a write guard, re-validating because another tick may have superseded/removed/resolved the target while unlocked. do_resolve's ANN search is the expensive part and runs read-only so daemon ops don't serialize behind one resolve.
- `do_persist` stale-write guard: if another writer (CLI direct-write, second daemon) advanced the store past what this graph reconciled to, the in-RAM kern may be older than the committed on-disk row and a per-kern overwrite would silently drop newer rows. It skips; the daemon's maintenance tick reloads via `reconcile_if_stale` and re-persists from the reconciled graph. Root persists through `merged_root` because authoritative fields (purpose/descriptors/radii) live on `graph.root`, not the map entry.
- `do_reembed`: the dirty flag is the durable source of truth (set on edit, cleared here once the vector is replaced). A dirty reason whose endpoint isn't embedded yet gets `nv == None`: it stays dirty for a later sweep rather than pinning a stale vector.
