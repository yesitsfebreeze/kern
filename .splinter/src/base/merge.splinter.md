# src/base/merge.rs — commentary

Second-pass migration:
- Module `//!`: the long lattice-op inventory (counters via GCounter join, heat/confidence via max, status via the Active<Superseded lattice, timestamps min-on-creation / max-on-activity) reduced to the contract line. Content-addressed ids are what make it sound: equal id ⇒ identical immutable content, so only mutable metadata ever joins.
- `join_superseded_by`: lexicographic order is deliberate and load-bearing. When two replicas each independently record a DIFFERENT superseding id (a genuine concurrent supersession), the join must pick the same winner on every replica or the graph diverges. String `>` is an arbitrary but TOTAL and replica-independent order, so it is a valid CRDT tiebreak — the winner carries no semantics, only determinism.
- `merge_entity` SECURITY (full rationale): a max-join is monotone-up and irreversible, so a compromised peer could pin a poisoned claim's confidence arbitrarily high federation-wide with no honest replica able to lower it (the confidence-by-max poisoning pin). Until inbound federation entities are trust/provenance-gated, confidence stays replica-local — derived only from this replica's own Bayesian observations (`observe_support` / `observe_contradict`). Replica-local values converge trivially (never shared), so the CRDT properties still hold. `unlinked_count` is local ingest bookkeeping, not convergent.
- `merge_remote_entity` (full rationale): the function must only ever touch `target_kern_id` and must NOT search other kerns for the id — otherwise a peer could forge an id colliding with a local-origin entity (or another network's) and CRDT-merge attacker-controlled metadata (status=Superseded, heat, confidence) into it, or repoint the global entity index at a forgery. The per-kern cap bounds a spamming peer.
- Match-arm labels (known shared content / id owned by another kern / unknown id) deleted — the arm patterns and the warn! messages already say it.
- `// Borrow of `kern` ends here; index via &mut self below.` deleted: a borrow-scope note the compiler enforces.
- test `merge_reason_maxes_score_and_joins_traversal_idempotently`: the deleted step labels covered idempotency (re-merge changes nothing) and monotonicity (a lower-score remote never lowers the score) — both are exercised, neither is in an assert message.
- test `superseded_by_join_picks_the_lexicographically_higher_id`: step narration deleted; the three cases are higher-sorting wins / lower-sorting loses / empty never clears.
- test `merge_entity_never_imports_replica_local_mutable_state`: kept only the field-addition-guard line. Full intent: a targeted stand-in for a full fuzz harness — poison EVERY replica-local field on the remote and assert none leak through the join, so a future field accidentally wired into `merge_entity` trips this test.
- test `merge_to_superseded_drops_entity_from_search_index`: the deleted essay restated the ANN-eviction invariant already stated at the `now_superseded` call site in `merge_remote_entity`.
