# src/base/types.rs — commentary

- `EntityStatus`: `Superseded` migrated here from the old `ThoughtKind` enum — kind and lifecycle used to be one enum; a superseded Fact keeps `kind = Fact`.
- `unnamed_kern_id` / `named_child_kern_id`: split out of the `Kern` ctors specifically so the id scheme is deterministic for a fixed `(parent_id, nonce)` and unit-testable without mocking the clock; `now_nanos()` isolates the single non-deterministic input behind one named call.

Second-pass migration:
- `EntityKind` variants: per-variant docs deleted as name restatements (Claim = default unverified statement; Document = source artifact — file body, ticket body, session slice, agent blob; Question = open inquiry awaiting an answer; Answer = resolution to a Question; Conclusion = synthesized stance over many Claims). Only `Fact`'s immutability gloss kept — it is a real invariant (`reason::remove_entity` makes Active Facts cascade-immune). The enum-level traps (`Receipt` is not a kind; `Superseded` moved to `EntityStatus`) stay inline.
- `EntityKind::parse` / `Source::parse_scheme`: docs deleted, the signature says it (`Option` = `None` on unknown input).
- `EntityKind::as_str` / `Source::scheme`: "stable" is the load-bearing word — the MCP query tool's `kind`/`scheme` filters match these strings, so they are an external contract, not a display detail.
- `Source` enum: the scheme table was flattened to one line. Grammar unchanged: `file://<path>`, `ticket://<system>/<id>[#section]`, `session://<id>[#slice]`, `agent://<name>`, `inline://<hash>`.
- `Entity::dirty` / `Reason::dirty`: a dirty entity has a stale `vector`/`gnn_vector` until the reevaluation sweep re-embeds it and clears the flag; it is persistent (`serde(default)`, not skip) specifically so an interrupted reevaluation resumes after restart. A dirty Reason's vector is recomputed as the mean of its endpoints.
- Bi-temporal fields (full rationale for `#[serde(skip)]`): keeping `valid_from`/`valid_to`/`invalidated_at` out of `Entity`'s bincode stream leaves the embedded on-disk/gossip layout byte-identical to pre-temporal snapshots — no wire break, no silent decode loss. The primary store persists them out-of-band via `StoredKern`'s temporal side-map (the same pattern as the int8 vector side-map). The cold tier and gossip carry only the always-serialized `status`/`superseded_by`, which is the essential "this is superseded" signal; the timestamps are a node-local refinement. Design lineage: Zep-style "supersede, don't delete".
- `Entity::set_text`: collapses the thought to a single Context chunk and drops the statement refs the original distillation produced — a real side effect, kept inline.
- `valid_from_or_created`: the explicit `valid_from` hint is what the distiller extracts from phrasing like "since March"; otherwise the lower bound is ingestion time.
- `now_nanos` / `unnamed_kern_id` / `named_child_kern_id`: docs compressed. The point of the split (already noted above) is that `now_nanos()` isolates the single non-deterministic input, leaving the two id derivations pure and unit-testable.
- `Kern::new_named_child`: dropped the "default radii come from the kern routing constants" line — the `KERN_INNER_RADIUS`/`KERN_OUTER_RADIUS` assignments are right there.
- `mk_entity`: used by tests across the base/retrieval/gossip modules; the point is that the 25-field `Entity` literal lives in exactly one place.
- test `conf_mean_and_variance_handle_a_zero_total_prior`: narration deleted — alpha = beta = 0 is a degenerate prior, mean falls back to 0.5 and variance to 0, which the asserts state. The Beta(2,1) derivation in the sibling test is KEPT inline as a 1-line magic-number justification (mean = a/(a+b) = 2/3; var = ab/((a+b)^2(a+b+1)) = 2/36).
- test `kern_id_derivation_is_deterministic_and_input_sensitive`: step labels deleted. The four properties it locks: deterministic for a fixed (parent, nonce) — the property the bare `SystemTime::now()` call previously made un-testable; the nonce disambiguates rapid successive creations under one parent; parent and anchor name both feed the hash; ids are non-empty content hashes.
